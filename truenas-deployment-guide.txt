# KAYO PORTAL - TRUENAS ARCHITECTURE & DEPLOYMENT GUIDE

## 1. PROJECT OVERVIEW
The "Portfolio" project (Kayo Portal) is a Next.js 14 application serving as your central Operations Hub and Developer Dashboard. It is NOT a static website. It acts as a dynamic Project Manager and Process Launcher for all your surrounding codebases. 

Whenever you click a project card in the Portal UI, the backend API dynamically spins up the required local development servers (`npm run dev`), hooks the server to an available network port, securely proxies traffic to it, and runs background automated screenshot routines.

## 2. HOW THE CORE FEATURES WORK

**a) Data Architecture (`config/projects.json`):**
The entire Dashboard is strictly data-driven. The `projects.json` array powers the UI grid and defines whether a project is "live," "building," or "offline." 

**b) The Dynamic Project Launcher (`app/api/launch/route.ts`):**
When a project card is clicked, this API route intercepts it.
- Resolves the absolute path to the sibling directory (e.g., `../clothing`).
- Spawns a detached background terminal process running `npm run dev` in that sibling directory.
- Binds the host strictly to `0.0.0.0` (critical for your Cloudflare Tunnel environment so external devices can reach the spawned port).
- Dynamically assigns an open port (`3001`, `3002`, etc.) and redirects the user’s browser there.

**c) The Automated Screenshot Engine (`scripts/dynamic-screenshot.js`):**
Right after the Launcher spins up your project, it invisibly executes this worker thread. 
- It waits 8-10 seconds for the internal framework (React/Vite) to compile.
- It scans the HTML for framework errors overlay screens.
- It triggers a headless Puppeteer browser, snaps a high-res screenshot, saves it to `public/projects/`, and patches `projects.json` to immediately reflect the new visual state on the Home Page.

**d) Repository Sync Engine (`scripts/sync.js`):**
A Node script accessible via the "Sync GitHub" UI button that securely hits the GitHub API to detect all isolated repositories you own and `git clone`s missing ones into your TrueNAS workspace automatically.

## 3. TRUENAS DEPLOYMENT CHECKLIST & WARNINGS

Because this system literally acts as an orchestrator over your file system and spawns child processes, deploying it to a TrueNAS container (like an App, Jail, or Docker instance) has some highly specific requirements:

### A. Directory Structure Parity (CRITICAL)
In your TrueNAS filesystem, the `Portfolio` folder MUST be mounted inside a parent `Workspace` folder containing all other repositories next to it.
```text
/mnt/storage/workspace/
├── Portfolio/       <-- (Runs Next.js on port 3000 mapped to Cloudflare)
├── clothing/        <-- (Spawned dynamically by Portfolio API on port 3001)
├── leads-website/   <-- (Spawned dynamically by Portfolio API on port 3002)
└── ...
```

### B. OS Executables Needed inside the TrueNAS Container
The container hosting the Portfolio must have the following system libraries installed, not just Node:
- **Node.js (v18+)** and `npm`.
- **Git** (Required for the "Sync GitHub" API route to run `git clone`).
- **Chromium / Linux Browser** (Required for the Puppeteer Screenshot script).

### C. The Puppeteer Path Problem (Must Fix Before Deploy)
Right now, `scripts/dynamic-screenshot.js` and `screenshots.js` are hardcoded to look for Microsoft Edge on Windows (`C:\\Program Files\\...`).
Before running on TrueNAS Linux, you MUST edit those scripts to point to the Linux browser binary (e.g., `/usr/bin/chromium` or `/usr/bin/google-chrome-stable`).

### D. Cloudflare Tunnels & Ports
Your Cloudflare Tunnel will point your subdomain (`portal.kayo.com`) to the TrueNAS local IP on port `3000`. 
**However**, remember that when the Launcher spins up the `clothing` app, it puts it on port `3001`. Cloudflare Tunnel won't know about port `3001` automatically! 
- **Solution:** You will either need to map wildcard subdomains through Cloudflare (e.g., `*.kayo.com` routes to dynamically assigned ports) or expose the entire port range (3001-3015) outward through your TrueNAS networking stack if you want to access the sub-projects externally.
